Olá!
Nessa aula você irá passar pela Preparação, que é um momento anterior 
as 5 etapas do Design Thinking. 
Para isso você precisará, no decorrer dessa aula, de algumas tabelas. 
Essas tabelas podem ser usadas no computador mesmo, 
em um editor de texto comum como o Drawn do Google Drive (Tabela feitas, não do melhor jeito, acho)
ou você pode pegar uma folha A4/A3 ou flipchart e desenhar a tabela a mão. 
Os títulos das tabelas que utilizaremos são os seguintes:

1) Problemas - Coisas que eu gostaria que tivesse; {
	//Aqui preenchemos o problema que temos ou os problemas; ou o que gostaríamos que existisse
	Gostaria que pessoas que trabalham em empresas conseguissem contratar um plano corporativo facilmente
	no site da Alura; /* Muito engessado */
	Gostaria que os alunos sempre tivessem os posts sempre respondidos no forum;
}

2) Como podemos; {
	//Aqui o problema de uma forma que uma possibilidade possa ser deslumbrada
	Desenvolver uma forma que uma empresa consiga contratar um plano de estudos no site da Alura de maneira facil; /* Vendo a possibilidade */
	Achar uma maneira que os alunos mais experientes sempre possam ajudar;
}

3) Seja simples; {
	//Aqui descrevemos o problema de maneira simples, objetiva e que qualquer pessoa consiga entender; às vezes a forma
	//como foi descrito no questionário 2, poder ser a mesma descrita aqui.
	Pessoas que precisam contratar planos corporativos não conseguem fazer isso pelo nosso site.
}

4) Medidas/Indicadores de Sucesso; {
	//Aqui descrevemos os indicadores que nos auxiliarão a perceber se fomos bem sucedidos
	//ao resolver o nosso problema escolhido do DT.
	Feedback positivo dos alunos.
	Acesso a informação desses planoes de maneira simples.
	Contratação desses planos de maneira simples e ágil.
	Aumento na demanda por este tipo de contratação.
}

5) Estabeleça limites; {
	//Limitações que devemos levar em consideração ao pensar nas soluções para o nosso problema
	Precisa ser de baixo custo.
	Precisa utilizar recursos humanos/intelectuais que já temos no Alura, que não necessite de 
	recursos externos (exemplo: uma peça de design? Temos designers aqui; Enviar uma carta?
	Teremos que enviar via correios, não é nosso)
}

6) Escreva um Resumo - Briefing; {
	//Exemplo dado pela instrutora
	Nossos futuros alunos não conseguem contratar planos que não sejam aqueles oferecidos em nosso site.
	Fernando conseguiu um budget para investir na formação da sua equipe de T.I. na empresa, mas não
	encontra como pode contratar um plano corporativo no site da Alura. Existe alguma maneira de
	solucionarmos isto? Porque para nós, da Alura, isso é um problema. Não queremos que alguém precise
	procurar tanto em nosso site para encontrar uma simples informação de um plano diferente ou uma
	solicitação de informação sobre um plano específico.
}

7) Objetivos finais; {
	//Onde queremos chegar?
	//Aqui falamos onde esperamos chegar ao fim do processo do design thinking,
	//descrevendo até mesmo qual o entregável podemos ter.
	/*
		Definir objetivos para empreender o desafio de design significa estabelecer onde esperamos chegar 
		ao fim do processo do Design Thinking, descrever até mesmo qual o entregável podemos ter no fim do processo. 
		Isso pode nos auxiliar durante todo o processo e até mesmo ser um termômetro durante todo o processo.
	*/
	Um documento do Google Drive para compartilhar com os demais tomadores de decisão
	Uma apresentação resumo do que foi estudado e das soluções as quais chegamos
}

Qualquer dúvida ou problema para criar essas tabelas, avise no forum!
